# Feature Requirements (Feats)

This folder contains formal requirements documentation for planned or in-progress features. Use these docs as the source of truth for implementation and acceptance testing.

## Document Structure

Feature docs typically include:

1. **Overview** — What the feature is and where it lives
2. **Current Behavior** — Baseline / problem statement
3. **Target Behavior** — Desired UX (with reference examples)
4. **Functional Requirements** — Numbered, prioritised requirements (Must/Should/Could)
5. **Interaction Matrix** — User action → expected result
6. **Edge Cases** — Special scenarios and expected behavior
7. **Technical Notes** — Backend/frontend considerations
8. **Acceptance Criteria** — Checklist for done
